42、任务与信号量：二进制信号量和计数信号量的使用

本课程详细讲解FreeRTOS中二进制信号量和计数信号量的概念、使用方法、应用场景以及它们之间的区别。

信号量概述

信号量是FreeRTOS中用于任务间同步和互斥的重要机制。它本质上是一个计数器，用于控制对共享资源的访问。

信号量的主要作用：

  • 任务同步：协调多个任务的执行顺序
  • 资源管理：控制对共享资源的访问
  • 事件通知：通知任务某个事件已经发生

信号量的两种类型：

  1. 二进制信号量：只能取0或1的信号量
  2. 计数信号量：可以取多个值的信号量

二进制信号量

二进制信号量是一种特殊的信号量，其计数值只能是0或1。它通常用于任务同步和互斥。

二进制信号量的特点：

  • 计数值只能是0或1
  • 常用于任务同步和互斥
  • 获取和释放操作会改变信号量的状态
  • 可以用于中断服务程序和任务之间的同步

二进制信号量的API函数：

  • xSemaphoreCreateBinary() - 创建二进制信号量
  • xSemaphoreTake() - 获取信号量
  • xSemaphoreGive() - 释放信号量
  • xSemaphoreGiveFromISR() - 在中断中释放信号量

二进制信号量使用示例

以下是一个使用二进制信号量进行任务同步的示例：

#include "FreeRTOS.h"
#include "task.h"
#include "semphr.h"

// 定义二进制信号量句柄
SemaphoreHandle_t xBinarySemaphore;

// 任务1：生产者任务
void vProducerTask(void *pvParameters)
{
    while(1)
    {
        // 执行一些操作...
        vTaskDelay(pdMS_TO_TICKS(1000));
        
        // 释放二进制信号量
        xSemaphoreGive(xBinarySemaphore);
    }
}

// 任务2：消费者任务
void vConsumerTask(void *pvParameters)
{
    while(1)
    {
        // 等待二进制信号量
        if(xSemaphoreTake(xBinarySemaphore, portMAX_DELAY) == pdTRUE)
        {
            // 收到信号量，执行相应操作
            // ...
        }
    }
}

int main(void)
{
    // 创建二进制信号量
    xBinarySemaphore = xSemaphoreCreateBinary();
    
    // 创建任务
    xTaskCreate(vProducerTask, "Producer", 1000, NULL, 1, NULL);
    xTaskCreate(vConsumerTask, "Consumer", 1000, NULL, 1, NULL);
    
    // 启动调度器
    vTaskStartScheduler();
    
    return 0;
}

计数信号量

计数信号量的计数值可以大于1，通常用于管理多个相同的资源。

计数信号量的特点：

  • 计数值可以大于1
  • 常用于资源池管理
  • 可以跟踪可用资源的数量
  • 允许多个任务同时访问资源

计数信号量的API函数：

  • xSemaphoreCreateCounting() - 创建计数信号量
  • xSemaphoreTake() - 获取信号量
  • xSemaphoreGive() - 释放信号量
  • xSemaphoreGiveFromISR() - 在中断中释放信号量

计数信号量使用示例

以下是一个使用计数信号量管理资源池的示例：

#include "FreeRTOS.h"
#include "task.h"
#include "semphr.h"

// 定义计数信号量句柄（假设有5个可用资源）
SemaphoreHandle_t xCountingSemaphore;

// 任务：使用资源的任务
void vResourceUserTask(void *pvParameters)
{
    while(1)
    {
        // 尝试获取资源（等待最多100ms）
        if(xSemaphoreTake(xCountingSemaphore, pdMS_TO_TICKS(100)) == pdTRUE)
        {
            // 成功获取资源，使用资源...
            // ...
            
            // 使用完成后释放资源
            xSemaphoreGive(xCountingSemaphore);
        }
        else
        {
            // 获取资源失败，执行其他操作或等待
            // ...
        }
        
        vTaskDelay(pdMS_TO_TICKS(500));
    }
}

int main(void)
{
    // 创建计数信号量，最大计数值为5，初始计数值为5
    xCountingSemaphore = xSemaphoreCreateCounting(5, 5);
    
    // 创建多个资源使用任务
    xTaskCreate(vResourceUserTask, "User1", 1000, NULL, 1, NULL);
    xTaskCreate(vResourceUserTask, "User2", 1000, NULL, 1, NULL);
    xTaskCreate(vResourceUserTask, "User3", 1000, NULL, 1, NULL);
    
    // 启动调度器
    vTaskStartScheduler();
    
    return 0;
}

二进制信号量与计数信号量对比

二进制信号量

特性 描述
计数值范围 0或1
主要用途 任务同步、互斥
创建函数 xSemaphoreCreateBinary()
适用场景 一对一或一对多的任务同步
资源管理 不适合管理多个相同资源

计数信号量

特性 描述
计数值范围 0到最大计数值
主要用途 资源池管理、流量控制
创建函数 xSemaphoreCreateCounting()
适用场景 管理多个相同的资源
资源管理 非常适合管理多个相同资源

信号量使用注意事项

常见问题与解决方案：

  • 优先级反转：使用互斥信号量而非二进制信号量进行互斥
  • 死锁：确保获取和释放信号量的顺序一致
  • 资源泄漏：确保每次获取后都释放信号量
  • 中断中使用：使用xSemaphoreGiveFromISR()在中断中释放信号量

最佳实践：

  1. 根据需求选择合适的信号量类型
  2. 为信号量设置合理的等待时间
  3. 在中断服务程序中只释放信号量，不获取信号量
  4. 确保信号量的获取和释放成对出现
  5. 使用信号量名称和注释提高代码可读性
