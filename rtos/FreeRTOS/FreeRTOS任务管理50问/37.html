37、任务就绪列表：任务就绪列表的工作原理

什么是任务就绪列表？

任务就绪列表是FreeRTOS内核中用于管理所有处于就绪状态任务的数据结构。它按照任务的优先级组织任务，确保调度器能够快速找到最高优先级的就绪任务来运行。

在FreeRTOS中，任务就绪列表使用以下关键数据结构：

  • pxReadyTasksLists：一个任务列表数组，每个优先级对应一个列表
  • uxTopReadyPriority：记录当前最高就绪优先级
  • uxCurrentNumberOfTasks：当前就绪任务数量
注意：FreeRTOS支持多优先级，相同优先级的任务采用时间片轮转调度。

就绪列表的数据结构

FreeRTOS使用位图和链表相结合的方式实现高效的就绪列表管理：

数据结构 作用 类型
pxReadyTasksLists 按优先级存储就绪任务链表 List_t[]
uxTopReadyPriority 记录最高就绪优先级 UBaseType_t
pxCurrentTCB 指向当前运行任务的TCB TCB_t*

关键代码定义：

/* 在tasks.c中定义 */
static List_t pxReadyTasksLists[ configMAX_PRIORITIES ];
static volatile UBaseType_t uxTopReadyPriority = tskIDLE_PRIORITY;

任务就绪列表工作原理

任务就绪列表的工作流程：

  1. 当任务创建或从阻塞状态恢复时，被添加到对应优先级的就绪列表
  2. 系统更新uxTopReadyPriority以反映当前最高就绪优先级
  3. 调度器通过查找位图快速确定最高优先级
  4. 从该优先级的就绪列表中选择下一个要运行的任务
  5. 如果使用时间片轮转，相同优先级任务按顺序执行

关键操作函数

FreeRTOS提供了一系列函数来操作任务就绪列表：

函数 功能
prvAddTaskToReadyList() 将任务添加到就绪列表
prvRemoveTaskFromReadyList() 从就绪列表移除任务
taskSELECT_HIGHEST_PRIORITY_TASK() 选择最高优先级任务
uxTaskPriorityGet() 获取任务优先级

示例代码：添加任务到就绪列表

void prvAddTaskToReadyList( TCB_t * const pxTCB )
{
    /* 将任务添加到对应优先级的就绪列表 */
    vListInsertEnd( &( pxReadyTasksLists[ pxTCB->uxPriority ] ), 
                    &( pxTCB->xStateListItem ) );
    
    /* 更新最高就绪优先级 */
    if( pxTCB->uxPriority > uxTopReadyPriority )
    {
        uxTopReadyPriority = pxTCB->uxPriority;
    }
}

调度器选择算法

FreeRTOS调度器使用高效的算法从就绪列表中选择任务：

  • 查找最高优先级：通过uxTopReadyPriority快速定位
  • 选择任务：从该优先级的就绪列表中选择任务
  • 时间片轮转：相同优先级任务轮流执行
  • 上下文切换：保存当前任务状态，恢复新任务状态

调度器核心代码逻辑：

void vTaskSwitchContext( void )
{
    /* 查找最高优先级就绪任务 */
    taskSELECT_HIGHEST_PRIORITY_TASK();
    
    /* 更新当前任务指针 */
    pxCurrentTCB = &pxReadyTasksLists[ uxTopReadyPriority ];
    
    /* 执行上下文切换 */
    portSWITCH_CONTEXT();
}
性能提示：FreeRTOS使用位图算法快速查找最高优先级，时间复杂度为O(1)，确保实时性。
