FreeRTOS任务通知发送：xTaskNotify和xTaskNotifyGive函数

第27课：深入理解任务通知机制及其发送函数

任务通知简介

任务通知是FreeRTOS提供的一种轻量级通信机制，每个任务都有一个32位的通知值。与队列、信号量等传统通信方式相比，任务通知具有以下优势：

  • 速度快：比队列、信号量等通信方式快45%
  • 内存占用少：每个任务只需额外占用8字节内存
  • 灵活性高：可以模拟二值信号量、计数信号量、事件组等多种通信方式

注意： 任务通知只能发送给特定任务，不能像队列或信号量那样由多个任务共享。每个任务只能有一个通知值，但可以通过不同的操作方式实现多种通信模式。

函数对比概览

xTaskNotify

功能： 向指定任务发送通知，可以更新任务的通知值，并可选地执行不同操作。

适用场景： 需要精确控制通知值更新的复杂通信场景。

特点： 功能强大，支持多种更新操作，但使用相对复杂。

xTaskNotifyGive

功能： 向指定任务发送通知，直接递增任务的通知值。

适用场景： 模拟计数信号量的简单通信场景。

特点： 使用简单，但功能相对有限。

xTaskNotify函数详解

函数原型

BaseType_t xTaskNotify( TaskHandle_t xTaskToNotify, 
                        uint32_t ulValue, 
                        eNotifyAction eAction );

参数说明

参数 类型 描述
xTaskToNotify TaskHandle_t 要接收通知的任务句柄
ulValue uint32_t 要传递的通知值
eAction eNotifyAction 指定如何更新任务的通知值

eAction参数选项

选项 描述
eNoAction 不更新任务的通知值，仅解除任务阻塞状态
eSetBits 将ulValue中的位设置到任务的通知值中（按位或操作）
eIncrement 递增任务的通知值（相当于xTaskNotifyGive）
eSetValueWithOverwrite 无条件覆盖任务的通知值为ulValue
eSetValueWithoutOverwrite 如果任务的通知值未被读取，则将其设置为ulValue

返回值

  • pdPASS：通知发送成功
  • pdFAIL：当使用eSetValueWithoutOverwrite且任务的通知值未被读取时返回

使用示例

// 示例1：设置特定位
#define TASK_BIT_0 (1UL << 0)
#define TASK_BIT_1 (1UL << 1)

// 设置任务通知值的第0位和第1位
xTaskNotify(xTaskHandle, TASK_BIT_0 | TASK_BIT_1, eSetBits);

// 示例2：递增通知值
xTaskNotify(xTaskHandle, 0, eIncrement);

// 示例3：覆盖通知值
xTaskNotify(xTaskHandle, 0x12345678, eSetValueWithOverwrite);

xTaskNotifyGive函数详解

函数原型

BaseType_t xTaskNotifyGive( TaskHandle_t xTaskToNotify );

参数说明

参数 类型 描述
xTaskToNotify TaskHandle_t 要接收通知的任务句柄

返回值

总是返回pdPASS，因为该函数总是成功发送通知。

功能说明

xTaskNotifyGive函数实际上是xTaskNotify(xTaskToNotify, 0, eIncrement)的简化版本。它直接将任务的通知值加1，并解除任务的阻塞状态（如果任务正在等待通知）。

使用示例

// 发送任务
void vSenderTask(void *pvParameters)
{
    TaskHandle_t xReceiverHandle = (TaskHandle_t)pvParameters;
    
    for(;;)
    {
        // 执行一些工作...
        
        // 发送通知给接收任务
        xTaskNotifyGive(xReceiverHandle);
        
        vTaskDelay(pdMS_TO_TICKS(1000));
    }
}

// 接收任务
void vReceiverTask(void *pvParameters)
{
    uint32_t ulNotificationValue;
    BaseType_t xResult;
    
    for(;;)
    {
        // 等待通知，最多等待100ms
        ulNotificationValue = ulTaskNotifyTake(pdTRUE, pdMS_TO_TICKS(100));
        
        if(ulNotificationValue > 0)
        {
            // 处理通知
            // ulNotificationValue表示接收到的通知次数
        }
        else
        {
            // 超时处理
        }
    }
}

函数对比分析

特性 xTaskNotify xTaskNotifyGive
功能复杂度 高（支持多种操作模式） 低（仅递增通知值）
使用简便性 较复杂 非常简单
灵活性 非常高 有限
适用场景 复杂通信、事件标志、数据传递 简单信号量模拟
性能 高 非常高

选择建议： 对于简单的计数信号量场景，优先使用xTaskNotifyGive；对于需要精确控制通知值或传递数据的复杂场景，使用xTaskNotify。

最佳实践与注意事项

使用建议

  1. 对于简单的信号量场景，优先使用xTaskNotifyGive和ulTaskNotifyTake组合
  2. 对于需要传递数据或事件标志的场景，使用xTaskNotify和xTaskNotifyWait组合
  3. 注意任务通知是"一次性"的，接收任务读取后通知值会被清除（取决于参数设置）
  4. 合理设置超时时间，避免任务永久阻塞

常见错误

  • 忘记处理xTaskNotify的返回值（特别是使用eSetValueWithoutOverwrite时）
  • 在中断服务程序中使用错误版本的函数（应使用带FromISR后缀的函数）
  • 混淆不同更新操作的行为，导致逻辑错误
  • 在多任务环境中未正确处理竞态条件

重要提示： 在中断服务程序中使用任务通知时，必须使用带FromISR后缀的函数，如xTaskNotifyFromISR和vTaskNotifyGiveFromISR，并在必要时进行上下文切换。
