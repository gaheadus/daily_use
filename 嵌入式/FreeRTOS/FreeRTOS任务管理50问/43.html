43、任务与互斥量：互斥量的使用和优先级继承

什么是互斥量？

互斥量（Mutex）是FreeRTOS中用于保护共享资源的一种同步机制。它确保在任何时刻只有一个任务可以访问被保护的资源，从而避免数据竞争和不一致。

关键特性：
  • 互斥量具有所有权概念
  • 支持优先级继承机制
  • 防止优先级反转问题
  • 递归互斥量允许同一任务多次获取

互斥量的创建

在FreeRTOS中，可以使用以下函数创建互斥量：

// 创建互斥量
SemaphoreHandle_t xMutex;

void setup() {
    // 创建互斥量
    xMutex = xSemaphoreCreateMutex();
    
    if (xMutex != NULL) {
        // 互斥量创建成功
    }
}

对于递归互斥量：

// 创建递归互斥量
SemaphoreHandle_t xRecursiveMutex;

xRecursiveMutex = xSemaphoreCreateRecursiveMutex();

互斥量的基本操作

函数 描述 返回值
xSemaphoreTake() 获取互斥量 pdTRUE/pdFALSE
xSemaphoreGive() 释放互斥量 pdTRUE/pdFALSE
xSemaphoreTakeRecursive() 递归获取互斥量 pdTRUE/pdFALSE
xSemaphoreGiveRecursive() 递归释放互斥量 pdTRUE/pdFALSE

互斥量使用示例

// 共享资源
int sharedCounter = 0;
SemaphoreHandle_t counterMutex;

void Task1(void *pvParameters) {
    while(1) {
        // 获取互斥量
        if(xSemaphoreTake(counterMutex, portMAX_DELAY) == pdTRUE) {
            // 临界区开始
            sharedCounter++;
            printf("Task1: Counter = %d\n", sharedCounter);
            // 临界区结束
            
            // 释放互斥量
            xSemaphoreGive(counterMutex);
        }
        vTaskDelay(1000 / portTICK_PERIOD_MS);
    }
}

void Task2(void *pvParameters) {
    while(1) {
        if(xSemaphoreTake(counterMutex, portMAX_DELAY) == pdTRUE) {
            sharedCounter--;
            printf("Task2: Counter = %d\n", sharedCounter);
            xSemaphoreGive(counterMutex);
        }
        vTaskDelay(1500 / portTICK_PERIOD_MS);
    }
}

优先级继承机制

优先级继承是互斥量的重要特性，用于解决优先级反转问题：

优先级反转场景：

  1. 低优先级任务获取互斥量
  2. 高优先级任务就绪，但需要等待互斥量
  3. 中优先级任务抢占低优先级任务
  4. 高优先级任务被中优先级任务阻塞

优先级继承解决方案：

  • 当高优先级任务等待互斥量时，低优先级任务临时继承高优先级
  • 低优先级任务尽快完成并释放互斥量
  • 高优先级任务获得互斥量并恢复原优先级

互斥量使用最佳实践

  • 保持临界区代码尽可能短
  • 不要在临界区内调用可能阻塞的函数
  • 确保获取和释放成对出现
  • 使用递归互斥量处理嵌套调用
  • 合理设置等待超时时间
  • 注意死锁情况的预防

常见问题与解决方案

问题 现象 解决方案
优先级反转 高优先级任务被低优先级任务阻塞 使用支持优先级继承的互斥量
死锁 任务相互等待对方释放资源 统一资源获取顺序，使用超时机制
资源饥饿 某些任务长时间无法获取资源 合理设置任务优先级，使用公平调度
