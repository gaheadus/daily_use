uxTaskPriorityGet函数概述

uxTaskPriorityGet()是FreeRTOS提供的一个核心API函数，用于查询指定任务的当前优先级。在复杂的嵌入式系统中，任务优先级可能会动态变化，了解任务的当前优先级对于系统调试和优化至关重要。

函数原型与参数说明

UBaseType_t uxTaskPriorityGet( TaskHandle_t xTask );
参数 类型 描述
xTask TaskHandle_t 要查询优先级的任务句柄。传入NULL表示查询当前调用任务
返回值 UBaseType_t 返回指定任务的当前优先级值

应用场景与使用示例

1. 查询自身任务优先级

void vTaskExample( void *pvParameters )
{
    // 获取当前任务的优先级
    UBaseType_t uxCurrentPriority;
    uxCurrentPriority = uxTaskPriorityGet( NULL );
    
    // 打印优先级信息
    printf("当前任务优先级: %d\n", uxCurrentPriority);
    
    for( ;; )
    {
        // 任务主体代码
        vTaskDelay( 1000 / portTICK_PERIOD_MS );
    }
}

2. 查询其他任务优先级

// 全局任务句柄声明
TaskHandle_t xDisplayTaskHandle;

void vDisplayTask( void *pvParameters )

// 创建任务时保存句柄
xTaskCreate( vDisplayTask, "Display", 1000, NULL, 2, &xDisplayTaskHandle );

使用注意事项

重要提示

  • uxTaskPriorityGet()函数可以在任务和中断服务程序(ISR)中调用
  • 在ISR中调用时，需要确保configUSE_TRACE_FACILITY配置为1
  • 查询不存在的任务句柄将导致未定义行为
  • 返回的优先级数值范围取决于configMAX_PRIORITIES配置

警告

  • 在多任务环境中，任务优先级可能在其他任务中被修改，获取的值可能不是最新值
  • 高优先级任务可能抢占当前任务，影响优先级获取的实时性
  • 避免在临界区内频繁调用此函数，以免影响系统性能

实际应用案例

动态优先级调整监控

// 任务优先级监控函数
void vTaskPriorityMonitor( void *pvParameters )
{
    TaskHandle_t xTargetTask = (TaskHandle_t)pvParameters;
    UBaseType_t uxLastPriority = 0;
    UBaseType_t uxCurrentPriority;
    
    for( ;; )
    {
        // 获取目标任务当前优先级
        uxCurrentPriority = uxTaskPriorityGet( xTargetTask );
        
        // 检测优先级变化
        if( uxCurrentPriority != uxLastPriority )
        {
            printf("任务优先级已改变: %d -> %d\n", 
                   uxLastPriority, uxCurrentPriority);
            uxLastPriority = uxCurrentPriority;
        }
        
        // 每隔1秒检查一次
        vTaskDelay( 1000 / portTICK_PERIOD_MS );
    }
}

// 创建优先级监控任务
xTaskCreate( vTaskPriorityMonitor, "PriMonitor", 512, 
             xTargetTaskHandle, 1, NULL );
